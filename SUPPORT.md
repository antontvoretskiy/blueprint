# Blueprint Support

This document explains what support is available for Blueprint and what belongs outside support.

## Supported Scope

Support covers:

- repository governance;
- Project Memory templates;
- recovery flow;
- Guardian checks;
- branch governance;
- Feature Lifecycle;
- PR Lifecycle;
- clean-start process;
- public documentation;
- example adoption patterns.

## Not Supported

Blueprint support does not cover:

- product implementation;
- application debugging;
- model behavior;
- prompt quality;
- deployment infrastructure;
- private project migration without sanitized evidence;
- runtime integrations that are not included in this repository.

## How To Ask For Help

Use GitHub issues for:

- documentation gaps;
- broken links;
- unclear template behavior;
- adoption questions;
- framework boundary questions.

Use the security policy for sensitive reports.

## Response Expectations

Blueprint is maintained as an open-source project. Response times are not guaranteed.

High-signal issues are easier to review when they include:

- repository shape;
- selected Blueprint installation shape;
- files being adapted;
- expected behavior;
- actual behavior;
- validation already run;
- sanitized examples.

## Maintainer Boundaries

Maintainers may close issues that:

- request product implementation;
- include private data;
- ask for unrelated application work;
- bypass the contribution standard;
- duplicate an existing answered issue.
