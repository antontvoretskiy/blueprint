## What To Record

Record work when it changes:

- public project status;
- implementation reality;
- architecture direction;
- governance rules;
- recovery state;
- source-of-truth ownership;
- release state;
- major templates, examples, or checklists.

## What Not To Record

Do not record:

- every small edit;
- every chat message;
- failed local attempts with no durable consequence;
- private product history that does not belong in this repository;
- PR details already captured in PR metadata unless they affect recovery.

## Recovery Use

Use this file to understand what changed over time and where to find the owner documents.

If current state differs from older task history, `08_CURRENT_STATE.md` wins for current recovery.
