# Blueprint Code Of Conduct

Blueprint should be a professional, direct, and safe open-source project for contributors.

## Expected Behavior

Contributors are expected to:

- communicate respectfully;
- keep discussion focused on the work;
- provide evidence for technical claims;
- respect maintainer scope decisions;
- keep private data out of public issues and PRs;
- follow the contribution standard.

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment or personal attacks;
- publishing private data;
- abusive or discriminatory language;
- repeated off-topic disruption;
- misrepresenting validation or authorship;
- pressuring maintainers to bypass security or review boundaries.

## Scope Decisions

Blueprint has strict product boundaries.

Maintainers may redirect or close contributions that add product implementation, runtime integrations, automation, or private project state outside the approved scope.

## Reporting

Report conduct concerns through the repository maintainer contact path.

Do not publish sensitive personal information in public issues.

## Enforcement

Maintainers may:

- request edits;
- hide or remove harmful content;
- close issues or PRs;
- limit participation when needed to protect the project.
